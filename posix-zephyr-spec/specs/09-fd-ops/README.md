# 文件描述符操作接口规范

## 概述

本模块定义POSIX文件描述符操作接口在Zephyr RTOS上的实现规范。

## 接口列表

1. `dup` - 复制文件描述符
2. `dup2` - 复制文件描述符到指定值
3. `fcntl` - 文件控制操作
4. `lseek` - 移动文件读写位置
5. `fsync` - 同步文件数据到存储
6. `fdatasync` - 同步文件数据（不含元数据）
7. `pipe` - 创建管道
8. `select` - I/O多路复用
9. `poll` - I/O多路复用（poll接口）

## 核心挑战

### 1. 文件描述符表管理
**问题**：Zephyr无系统级文件描述符表，需要模拟。

**解决方案**：
- 实现全局或线程级FD表
- 支持FD的分配、复制和释放
- 处理标准输入输出（0, 1, 2）的特殊映射

### 2. fcntl命令支持
**问题**：fcntl包含大量命令（F_GETFL, F_SETFL等），Zephyr驱动不直接支持。

**解决方案**：
- 实现命令分发器
- 模拟文件状态标志（O_NONBLOCK, O_APPEND等）
- 在FD表中维护标志状态

### 3. 文件偏移量管理
**问题**：Zephyr文件系统API需要显式传递偏移量。

**解决方案**：
- 在FD表中记录每个文件的当前偏移
- lseek修改偏移量
- read/write自动更新偏移

### 4. 管道实现
**问题**：Zephyr无内核级管道支持。

**解决方案**：
- 使用消息队列或环形缓冲区模拟
- 实现阻塞和非阻塞读写
- 处理管道满和空的边界条件

### 5. I/O多路复用
**问题**：select/poll需要监控多个FD的就绪状态。

**解决方案**：
- 实现FD就绪状态轮询机制
- 使用内核信号量实现超时等待
- 支持读、写、异常事件监控

## 优先级

1. **高优先级**：lseek, fcntl（基础功能）
2. **中优先级**：dup, dup2, fsync（常用功能）
3. **低优先级**：pipe, select, poll（高级功能）
