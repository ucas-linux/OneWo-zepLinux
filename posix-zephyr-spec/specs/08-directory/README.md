# 目录操作接口规范

## 概述

本模块定义POSIX目录操作接口在Zephyr RTOS上的实现规范。

## 接口列表

1. `opendir` - 打开目录
2. `readdir` - 读取目录项
3. `closedir` - 关闭目录
4. `rewinddir` - 重置目录读取位置
5. `mkdir` - 创建目录
6. `rmdir` - 删除目录
7. `getcwd` - 获取当前工作目录
8. `chdir` - 改变当前工作目录

## 核心挑战

### 1. 目录流管理
**问题**：Linux使用DIR*指针管理目录流，Zephyr使用fs_dir_t结构。

**解决方案**：
- 建立DIR到fs_dir_t的映射层
- 维护全局目录句柄表
- 实现目录流状态跟踪

### 2. 目录项结构转换
**问题**：struct dirent字段与Zephyr fs_dirent不完全对应。

**解决方案**：
- 映射d_name、d_type等关键字段
- 填充d_ino（inode号）为模拟值
- 处理长文件名截断问题

### 3. 工作目录概念
**问题**：Zephyr无进程概念，无传统意义的"当前工作目录"。

**解决方案**：
- 为每个线程维护工作目录上下文
- 使用线程本地存储（TLS）保存路径
- 实现路径解析时的相对路径转换

### 4. 递归目录操作
**问题**：嵌入式存储空间有限，递归操作需要栈管理。

**解决方案**：
- 限制目录深度
- 使用迭代而非递归实现
- 提供栈溢出保护

### 5. 目录遍历性能
**问题**：Flash存储读取速度慢，目录遍历可能阻塞。

**解决方案**：
- 实现目录项缓存
- 支持异步目录读取
- 优化文件系统索引结构

## 优先级

1. **高优先级**：opendir, readdir, closedir, mkdir（基础功能）
2. **中优先级**：rmdir, getcwd, chdir（常用功能）
3. **低优先级**：rewinddir（辅助功能）
