# fork - 创建子进程

## POSIX标准定义

```c
#include <unistd.h>

pid_t fork(void);
```

## 功能描述

创建一个新进程（子进程），它是调用进程（父进程）的副本。子进程继承父进程的内存空间、文件描述符、信号处理等。

## 返回值

- **父进程**：返回子进程的PID（>0）
- **子进程**：返回0
- **错误**：返回-1，设置errno

## Linux标准行为

1. 子进程获得父进程的内存副本（使用COW优化）
2. 子进程继承：
   - 打开的文件描述符
   - 信号处理设置
   - 当前工作目录
   - 环境变量
   - 资源限制
3. 子进程不继承：
   - 进程ID和父进程ID
   - 文件锁
   - 挂起的信号
   - 定时器

## Zephyr兼容性与退化行为

### 结论

在**无MMU**的Zephyr目标（如Cortex-M3）上，`fork()`无法提供Linux语义，必须返回错误。

### 行为规定

- **无MMU目标**：`fork()`返回`-1`并设置`errno=ENOSYS`。
- **具备MMU的目标**：可实现受限`fork`，但必须保证父/子地址空间隔离；如无法满足，返回`ENOSYS`。

### 测试要求

- 无MMU目标：验证`fork()`返回`-1`且`errno=ENOSYS`。
- MMU目标：验证父/子地址空间隔离与文件描述符继承（若支持）。
