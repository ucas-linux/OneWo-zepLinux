# 用户和组管理接口规范

## 概述

本模块定义POSIX用户和组管理接口在Zephyr RTOS上的实现规范。

## 接口列表

1. `getuid` - 获取用户ID
2. `geteuid` - 获取有效用户ID
3. `getgid` - 获取组ID
4. `getegid` - 获取有效组ID
5. `setuid` - 设置用户ID
6. `setgid` - 设置组ID
7. `getpwuid` - 通过UID获取用户信息
8. `getpwnam` - 通过用户名获取用户信息
9. `getgrgid` - 通过GID获取组信息
10. `getgrnam` - 通过组名获取组信息

## 核心挑战

### 1. 权限模型简化
**问题**：Cortex-M3无MMU，无法实现完整的Unix多用户权限隔离。

**解决方案**：
- 实现简化的用户/组概念
- 所有接口返回固定值（如UID=0, GID=0）
- 保留接口兼容性但不强制权限检查

### 2. 用户数据库模拟
**问题**：Zephyr无/etc/passwd和/etc/group文件。

**解决方案**：
- 使用静态数组或配置文件定义用户和组
- 实现简单的查找函数
- 支持编译时配置用户列表

### 3. 特权操作处理
**问题**：setuid/setgid在Linux中需要root权限。

**解决方案**：
- 在单用户模式下允许所有操作
- 或实现简单的权限检查（基于当前UID）
- 记录操作日志用于审计

### 4. 线程本地用户上下文
**问题**：多线程环境下每个线程可能需要不同的用户身份。

**解决方案**：
- 使用线程本地存储（TLS）保存UID/GID
- 实现线程级的用户上下文切换
- 处理线程创建时的身份继承

### 5. 与文件系统权限集成
**问题**：用户/组ID需要与文件权限检查配合。

**解决方案**：
- 在文件操作时检查当前线程的UID/GID
- 实现简化的权限检查逻辑
- 支持超级用户（UID=0）绕过检查

## 文件组织

- `getuid.md` - 用户ID获取详细规范
- `setuid.md` - 用户ID设置详细规范
- `passwd.md` - 用户数据库详细规范
- `permissions.md` - 权限检查详细规范
- `types.md` - 数据类型定义
- `implementation.md` - 实现指导

## 优先级

1. **高优先级**：getuid, getgid（基础功能）
2. **中优先级**：geteuid, getegid, setuid, setgid（常用功能）
3. **低优先级**：getpwuid, getpwnam, getgrgid, getgrnam（辅助功能）
