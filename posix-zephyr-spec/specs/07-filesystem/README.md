# 文件系统接口规范

## 概述

本模块定义POSIX文件系统操作接口在Zephyr RTOS上的实现规范。

## 接口列表

1. `mount` - 挂载文件系统
2. `umount` - 卸载文件系统
3. `stat` - 获取文件状态
4. `fstat` - 通过文件描述符获取状态
5. `access` - 检查文件访问权限
6. `chmod` - 修改文件权限
7. `chown` - 修改文件所有者
8. `truncate` - 截断文件
9. `ftruncate` - 通过文件描述符截断文件
10. `sync` - 同步文件系统缓存到存储设备

## 核心挑战

### 1. 文件系统类型适配
**问题**：Linux支持多种文件系统（ext4, FAT, NFS等），Zephyr主要支持FAT和LittleFS。

**解决方案**：
- 建立文件系统类型映射表
- 将Linux文件系统类型转换为Zephyr支持的类型
- 提供统一的挂载接口封装

### 2. 权限模型简化
**问题**：Cortex-M3无MMU，无法实现完整的Unix权限模型。

**解决方案**：
- 简化权限检查为只读/读写两级
- chmod/chown接口保留但仅记录元数据
- 在文件系统层模拟基本权限位

### 3. 文件状态结构映射
**问题**：Linux struct stat包含大量字段，Zephyr fs_dirent结构简化。

**解决方案**：
- 扩展Zephyr文件系统元数据存储
- 映射必要字段（大小、时间戳、类型）
- 不支持的字段返回合理默认值

### 4. 同步机制
**问题**：嵌入式存储需要显式同步以防数据丢失。

**解决方案**：
- 实现sync接口调用底层fs_sync
- 在关键操作后自动触发同步
- 提供配置选项控制同步频率

### 5. 挂载点管理
**问题**：Zephyr挂载点需要预定义，Linux支持动态挂载。

**解决方案**：
- 维护全局挂载点表
- 支持运行时注册挂载点
- 实现路径解析到挂载点的映射

## 优先级

1. **高优先级**：mount, umount, stat, fstat（基础功能）
2. **中优先级**：access, truncate, sync（常用功能）
3. **低优先级**：chmod, chown（权限管理）
