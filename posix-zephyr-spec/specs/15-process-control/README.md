# 进程控制接口规范

## 概述

本模块定义POSIX进程控制接口在Zephyr RTOS上的实现规范。

## 接口列表

1. `getpid` - 获取进程ID
2. `getppid` - 获取父进程ID
3. `wait` - 等待子进程结束
4. `waitpid` - 等待指定子进程
5. `system` - 执行shell命令
6. `abort` - 异常终止进程
7. `atexit` - 注册退出处理函数
8. `_exit` - 立即终止进程

## 核心挑战

### 1. 进程ID模拟
**问题**：Zephyr无进程概念，仅有线程。

**解决方案**：
- 将主线程视为"进程"
- 使用线程ID作为PID
- 维护父子线程关系表

### 2. 进程层次结构
**问题**：Zephyr线程无父子关系。

**解决方案**：
- 在线程创建时记录父线程ID
- 实现进程树数据结构
- 支持getppid查询父进程

### 3. 子进程等待机制
**问题**：wait/waitpid需要获取子进程退出状态。

**解决方案**：
- 扩展线程控制块保存退出状态
- 使用信号量实现父线程阻塞
- 维护僵尸进程列表（已退出但未回收）

### 4. system命令执行
**问题**：Zephyr无shell或命令解释器。

**解决方案**：
- 集成Zephyr Shell子系统
- 实现命令解析和执行
- 返回命令执行结果

### 5. 退出处理函数链
**问题**：atexit需要在进程退出时调用注册的函数。

**解决方案**：
- 维护全局或线程级的退出函数链表
- 在exit/_exit时按注册顺序逆序调用
- 处理退出函数中的异常

### 6. 资源清理
**问题**：进程退出时需要清理所有资源。

**解决方案**：
- 遍历并释放线程持有的所有内核对象
- 关闭打开的文件描述符
- 释放动态分配的内存
- 通知父进程并更新状态

## 文件组织

- `getpid.md` - 进程ID获取详细规范
- `wait.md` - 子进程等待详细规范
- `system.md` - 命令执行详细规范
- `atexit.md` - 退出处理详细规范
- `process_tree.md` - 进程树管理详细规范
- `types.md` - 数据类型定义
- `implementation.md` - 实现指导

## 优先级

1. **高优先级**：getpid, getppid, _exit（基础功能）
2. **中优先级**：wait, waitpid, atexit（进程管理）
3. **低优先级**：system, abort（辅助功能）
